Дорожная карта

Основная задача модуля «Дорожная карта» предоставить интерфейс представителям университетов, который позволит ввести и структурировать данные о дорожных картах в рамках программы «Приоритет 2030». На Рисунке № 1 представлена схема описывающая основные элементы и пользовательские роли модуля.

Рисунок №1 — Основные элементы и пользователи

Для перехода в интерфейс модуля «Дорожная карта» в левом меню личного кабинета участника программы «Приоритет 2030» выберите пункт «Приоритет2030», далее подпункт «Дорожная карта» (откроется «Главная страница модуля»).

Главная страница модуля​

На главной странице модуля «Дорожная карта» (см. Рисунок № 2) представлен список всех стратегических и институциональных проектов университета.

Рисунок № 2 — Главная страница модуля «Дорожная карта»

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Проекты» — по нажатии пользователю раскрывается/закрывается список с созданными проектами в рамках конкретного стратегического/институционального проекта.
  2. Иконка «Просмотр» — по нажатии пользователь перейдёт на страницу с описанием проекта
  3. Иконка «Редактировать» — по нажатии пользователь перейдёт на страницу редактирования проекта
  4. Иконка «Удалить» — по нажатии произойдёт попытка удалить проект (обратите внимание, система не позволит удалить проект, когда в нём есть хоть одна задача)
  5. Кнопка «Подробнее» — по нажатии пользователь перейдёт на страницу стратегического/институционального проекта

Стратегический/институциональный проект​

Рисунок № 3 — Информация о стратегическом/институциональном проекте

Обратите внимание

Страницы с информацией о стратегическом проекте отличаются от страниц с институциональными проектами в сторону уменьшения возможностей, о которых идёт речь далее по тексту (с явной пометкой об этом).

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии пользователь перейдёт на главную страницу дорожной карты
  2. Кнопка «Назначить» — по нажатии на кнопку пользователю будет предложено выбрать ответственного за стратегический/институциональный проект из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  3. Кнопка «Редактировать стратегический проект» — по нажатии пользователь перейдёт на страницу редактирования стратегического проекта (обратите внимание, что аналогичный элемент интерфейса отсутствует на страницах институциональных проектов).
  4. Вкладка «Основное» — по нажатии пользователь перейдёт к основной информация о стратегическом проекте (обратите внимание, аналогичная по смыслу вкладка в институциональных проектах называется «Описание»)
  5. Вкладка «Политики» — по нажатии пользователь перейдёт к таблице со списком политик, в столбцах которой указано: политика; влияние; наименование проекта (обратите внимание, вкладка отсутствует в институциональных проектах).
  6. Вкладка «Ожидаемые результаты» — по нажатии пользователь перейдёт к таблице со списком результатов, в столбцах которой указано: вид результата; описание; наименование задачи. Список формируется на основании данных введённых в проекты данного стратегического/институционального проекта.
  7. Вкладка «Показатели» — по нажатии пользователь перейдёт к таблице со списком показателей университета в рамках «Приоритет 2030», в столбцах которой указано: показатель; влияние. Влияние на показатель определяется исходя из введённых данных при редактировании стратегического проекта (обратите внимание, вкладка отсутствует в институциональных проектах, как и возможность редактирования).
  8. Вкладка «Проекты» — по нажатии пользователь перейдёт к списку проектов в рамках текущего стратегического/институционального проекта. Вкладка содержит большой функционал по работе с проектами, который описан ниже.
  9. Кнопка «Скачать» — по нажатии пользователь перейдёт к скачиванию файла прикреплённого к страт. проекту.

Вкладка «Проекты»​

Под выбранной вкладкой «Проекты» на странице конкретного стратегического/институционального проекта выводится полный перечень связанных проектов.

Рисунок № 4 — Содержание вкладки «Проекты»

На вкладке пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Добавить» — по нажатии пользователь перейдёт на страницу для ввода данных нового проекта.
  2. Поле «Поиска» — c помощью данного поля пользователь можете отфильтровать список проектов (осуществляется полнотекстовый поиск по названиям проектов).
  3. Кнопка «Подробнее» — по нажатии пользователь перейдёт на страницу с информацией о проекте.
  4. Кнопка «Редактировать» — по нажатии пользователь перейдёт на страницу редактирования информации о проекте.
  5. Кнопка «Удалить» — по нажатии пользователь произведёт попытку удалить (обратите внимание, система не позволит удалить проект, когда в нём есть хоть одна задача).

Редактирование стратегического проекта​

Рисунок № 5 — Редактирование информации о стратегическом проекте

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии пользователь перейдёт на страницу соответствующего стратегического проекта.
  2. Блок «Плановая реализация» — содержит два поля, позволяющих указать временные рамки стратегического проекта.
  3. Блок «Показатели» — в данном блоке пользователю предоставляется возможность выбрать уровень влияния стратегического проекта на показатели программы «Приоритет 2030».
  4. Блок «Файлы» — в данном блоке пользователь может прикрепить файлы к стратегическому проекту.
  5. Иконка «Удалить» — по нажатии удалится файл, на против которого размещена иконка.
  6. Кнопка «Добавить файл» — по нажатии в блок «Файлы» добавится новые элементы интерфейса (поле для указания типа файла и поле для прикрепления файла) позволяющие добавить новый файл.
  7. Кнопка «Сохранить» — по нажатии произойдёт сохранение даных указанных на странице.

Информация о проекте​

Рисунке № 6 — Страница с информацией о проекте

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии пользователь перейдёт на страницу с информацией о родительском стратегическом/институциональном проекте.
  2. Кнопка «Назначить» — по нажатии на кнопку пользователю будет предложено выбрать ответственного за проект из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  3. Кнопка «Редактировать» — по нажатии пользователь перейдёт на страницу редактирования проекта, на которой можно указать основную информацию о проекте.
  4. Вкладка «Основное» — по нажатии пользователь перейдёт к основной информации о проекте.
  5. Вкладка «Проектная команда» — по нажатии пользователь перейдёт к элементам интерфейса, позволяющим работать с членами команды проекта.
  6. Вкладка «Цели» — по нажатии пользователь перейдёт к таблице со списком целей проекта.
  7. Вкладка «Политики» — по нажатии пользователь перейдёт к таблице со списком политик, в столбцах которой указано: политика; влияние; наименование задач.
  8. Вкладка «Результаты» — по нажатии пользователь перейдёт к таблице со списком результатов, в столбцах которой указано: вид результата; описание; наименование задачи.
  9. Вкладка «Задачи» — по нажатии пользователь перейдёт к списоку задач в рамках текущего проекта. Вкладка содержит элементы интерфейса по работе с задачами.
Обратите внимание

Отображаемые данные во кладках «Цели», «Политики», «Результаты» проектов формируются на основании данных указанных пользователями в задачах, на соответствующих страницах и вкладках.

Вкладка «Проектная команда»​

Рисунок № 7 — Содержание вкладки «Проектная команда»

На вкладке пользователю доступны следующие элементы интерфейса:

  1. Иконка «Удалить» — позволяет удалить члена проектной команды из данного проекта.
  2. Поле «Участник» — позволяет сделать чатсью проектной команды одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  3. Поле «Роль» — позволяет ввести (произвольный текст) роль участника проектной команды.
  4. Кнопка «Сохранить» — по нажатии указанные данные об участнике команды (пункты 2 и 3 данного списка) добавятся в таблицу с общим списком проектной команды проекта.

Вкладка «Задачи»​

Рисунок № 8 — Содержание вкладки «Задачи»

На вкладке пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Добавить задачу» — по нажатии пользователь перейдёт на страницу для ввода данных новой задачи.
  2. Поле «Поиска» — с помощью данного поля пользователь можете отфильтровать список задач (осуществляется полнотекстовый поиск по названиям задач).
  3. Иконка «Просмотр» — по нажатии пользователь перейдёт на страницу с описанием задачи.
  4. Иконка «Редактировать» — по нажатии пользователь перейдёт на страницу редактирования задачи.
  5. Иконка «Удалить» — по нажатии произойдёт удаление задачи.

Создание/редактирование проекта​

Рисунок № 9 — Страница создания/редактирования проекта

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии пользователь перейдёт на страницу соответствующего стратегического проекта.
  2. Поле «Наименование» — позволяет указать (произвольный текст) название проекта.
  3. Поле «Описание» — позволяет указать (произвольный текст) описание проекта.
  4. Поле «Ответственный» — позволяет выбрать ответственного за проект из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  5. Поле «Вид проекта» — позволяет указать тип проекта.
  6. Блок «Плановая реализация» — содержит два поля, позволяющих указать временные рамки проекта. Важно помнить, что возможный диапазон ограничен датами указанными в родительском стратегическом/институциональном проекте.
  7. Блок «Цели» — позволяет работать с данными о целях проекта.
  8. Иконка «Удалить» — по нажатии цель удалится из блока «Цели».
  9. Кнопка «Добавить цель» — по нажатии в блок «Цели» добавится новое поле для указание названия новой цели.
  10. Блок «Файлы» — в данном блоке пользователь может прикрепить файлы к проекту.
  11. Иконка «Удалить» — по нажатии удалится файл, на против которого размещена иконка.
  12. Кнопка «Добавить файл» — по нажатии в блок «Файлы» добавится новые элементы интерфейса (поле для указания типа файла и поле для прикрепления файла) позволяющие добавить новый файл.
  13. Кнопка «Создать/Сохранить» — по нажатии произойдёт сохранение данных указанных на странице проекта.
Обратите внимание

Для научных и образовательных проектов, а также проектов с типом предпринимательство и инновации потребуется указать дополнительную информацию (точный список см. ниже на данной странице).

Страница с информацией о задаче​

Рисунок № 10 — Страница с информацией о задаче

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии пользователь перейдёт на страницу родительского проекта.
  2. Кнопка «Назначить» — по нажатии на кнопку пользователю будет предложено выбрать ответственного за задачу из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  3. Кнопка «Редактировать задачу» — по нажатии пользователь перейдёт на страницу редактирования задачи.
  4. Вкладка «Основное» — по нажатии пользователь перейдёт к основной информации о задаче.
  5. Вкладка «Участники консорциума» — по нажатии пользователь перейдёт к таблице со списком, в котором указано: название консорциума; наименование участника консорциума; вклад в реализацию задачи.

Создание/редактирование задачи​

Рисунок № 11 — Интерфейс создания/редактирования задачи

На странице пользователю доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии, в случае создания новой задачи, пользователь перейдёт на страницу родительского проекта, а в случае редактирования задачи, на страницу с информацией о задаче.
  2. Поле «Наименование» — позволяет указать (произвольный текст) название задачи.
  3. Поле «Описание» — позволяет указать (произвольный текст) описание задачи.
  4. Поле «Ответственный» — позволяет выбрать ответственного за задачу из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  5. Блок «Плановая реализация» — содержит два поля, позволяющих указать временные рамки задачи. Важно помнить, что возможный диапазон ограничен датами указанными в родительском проекте.
  6. Поле «Введите имя участника консорциума или выберите из списка» — позволяет пользователю выбрать участника консорциума, одновременно с самим консорциумом.
  7. Поле «Вклад в реализацию задачи» — позволяет пользователю указать (произвольный текст) вклад участника консорциума в реализацию текущей задачи.
  8. Иконка «Удалить» — по нажатии удалится участник консорциума из блока «Участники консорциума».
  9. Галочка «Участника консорциума нет в списке» — элемент интерфейса скрывает поле «Введите имя участника консорциума или выберите из списка» и добавляет поля «ИНН участника» и «Полное наименование участника» для указания компании, отсутствующей в консорциумах университета.
  10. Кнопка «Добавить участника консорциума» — по нажатии в блок «Участники консорциума» добавляются элементы описанные в пунктах 6-9 данного списка, для добавления нового участника консорциума связанного с задачей.
  11. Поле «Вид результата» — позволяет выбрать из конечного списка один из видов результата.
  12. Поле «Ожидаемый результат» — позволяет описать (произвольный текст) ожидаемый результат.
  13. Поле «Политика» — позволяет выбрать одну из политик.
  14. Поле «Влияние» — позволяет выбрать уровень влияния на политику, выбранную с помощью элемента интерфейса описанного в предыдущем пункте данного списка.
  15. Кнопка «Создать/Сохранить» — по нажатии произойдёт сохранение данных указанных на странице задачи.
