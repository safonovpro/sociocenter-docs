Планируемая деятельность

Общая концепция​

Раздел «Планируемая деятельность» является особым в заявке университета и самым сложным с точки зрения интерфейсов и количества данных, которые необходимо указать университетам. Основная задача раздела — всесторонне описать передовую инженерную школу университета:

  • образовательные программы (ОП) и научные проекты (НП), на которых они базируются ОП. Важно, что количество связей ОП с НП никак не регламентировано и не ограничено, но именно на базе тематик НП будут выполняться тематические группировки ПИШ университетов (например для экспертов и комиссии).
  • связать НП и ОП с партнёрами университета.
  • сформировать команды и закрепить руководителей НП и ОП.
  • составить перечень критически важного оборудования и программного обеспечения, необходимого для НП и ОП.

Рисунок № 1 — Общая концепция

Обратите внимание

В отличии от всех остальных разделов личного кабинета, доступ к сущностям/страницам раздела «Планируемая деятельность» определяется не ролями и доступами в рамках системы (более подробнее читайте в соответствующем разделе), а назначением руководителя ПИШ и ответственных в рамках НП и/или ОП.

При переходе в раздел пользователю становятся доступны интерфейсы проиллюстрированные на Рисунке № 2 и 3, содержащие список научных проектов и образовательных программ университета в рамках ПИШ.

Рисунок № 3 — Вкладка «Образовательные программы»

На Рисунках 2 и 3 пользователю доступны следующие элементы интерфейса:

  1. Иконка «Редактировать» — по нажатии пользователь открывается интерфейс для редактиварония пользователя.
  2. Кнопка «Назначить» — по нажатии на кнопку пользователю будет предложено выбрать руководителя ПИШ из одного из зарегистрированных пользователей системы, связанных с университетом (более подробнее о работе с пользователями читайте по ссылке). Обратите внимание, что управление исчерпывающей информацией о руководителе ПИШ происходит в специальном разделе.
  3. Блок вкладок — позволяет пользователю переключаться между списками НП и ОП.
  4. Кнопка «Добавить» — позволяет добавить новый НП или ОП. Интерфейс добавления новых научных проектов и образовательных программ полностью повторяют интерфейс редактирования соответствующих сущностей.
  5. Строка поиска — осуществляет полнотекстовый поиск по названием из списка НП или ОП.
  6. Кнопка «Подробнее» — по нажатии пользователь перейдёт на страницу просмотра научного проекта или образовательной программы.
  7. Кнопка «Редактировать» — по нажатии пользователь перейдёт на страницу редактирования основной информации научного проекта или образовательной программы.
  8. Кнопка «Удалить» — удалит из списка НП или ОП.

Научные проекты​

Рисунок № 4 — Основная информация о НП

На странице доступны следующие элементы интерфейса:

  2. Иконка «Редактировать» — по нажатии пользователь открывается интерфейс для редактиварония пользователя.
  3. Кнопка «Назначить» — по нажатии на кнопку пользователю будет предложено выбрать ответственного или научного руководителя НП из одного из зарегистрированных пользователей системы (более подробнее о работе с пользователями читайте по ссылке), связанных с университетом.
  4. Кнопка «Редактировать» — по нажатии на кнопку пользователь перейдёт к странице с редактирование основной информации о научном проекте, которая отображается во вкладке «Основное» на данной странице.
  5. Блок вкладок — позволяет переключаться между следующими типами информации:
    • Основная информация
    • Партнёры
    • Оборудование и программное обеспечение
    • Проектная команда

Редактирование основного о НП​

Рисунок № 5 — Редактирование основной информации о НП

На странице доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии на кнопку пользователь перейдёт к основной информации НП.
  2. Поле «Название» — текстовое поле для указания название НП.
  3. Поле «Описание проекта» — текстовое поле для описания НП.
  4. Поле «Описание результата» — текстовое поле для описания результатов выполнения НП.
  5. Поле «Рубрика ГРНТИ» — поле для выбора одного из значений справочника «Государственный рубрикатор научно-технической информации».
  6. Поле «Классификатор OECD» — поле для выбора одного из значений справочника «Классификатор OECD»
  7. Поле «Связь с Приоритетами СНТР» — поле для выбора одного из значений справочника «Приоритеты стратегии научно-технологического развития Российской Федерации»
  8. Поле «Связь с Большими вызовы СНТР» — поле для выбора одного из значений справочника «Большие вызовы стратегии научно-технологического развития Российской Федерации»
  9. Поле «Код РНФ» — поле для выбора одного из значений справочника «Классификатор российского научного фонда»
  10. Поле «Критическая технология» — поле для выбора одного из значений справочника «Перечень критических технологий Российской Федерации»
  11. Поле «РАН развития науки, технологий и техники РФ» — поле для выбора одного из значений справочника
  12. Поле «Рубрикатор РАН» — поле для выбора одного из значений справочника
  13. Поле «От» — поле для указания даты начала проекта
  14. Поле «До» — поле для указания даты окончания проекта
  15. Чекбокс «Результаты научного проекта имеют предпринимательский и инновационный потенциал» — позволяет обозначить пользователю, что результаты выполнения проекта результаты имеют соответствующий потенциал.
  16. Чекбокс «Запланировано создание новых научных пространств» — позволяет обозначить пользователю, что в рамках выполнения НП запланировано создание новых научных пространств.
  17. Кнопка «Сохранить» — по нажатии указанные данные научного проекта сохраняются, а пользователь будет направлен на страницу с информацией о НП.

Редактирование основного об ОП​

Рисунок № 7 — Редактирование основной информации об ОН

На странице доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии на кнопку пользователь перейдёт к основной информации ОП.
  2. Поле «Название образовательной программы» — текстовое поле для указания название ОП.
  3. Поле «Специальность и направления подготовки» — поле для выбора специальности и/или направления подготовки в соответствии с нормативными документами конкурса.
  4. Поле «Тип программы» — поле для выбора типа программы.
  5. Поле «Количество задействованных полных ставок»* — поле для указания количества задействованных ставок штатных сотрудников университета.
  6. Поле «Количество полных ставок совместителей»* — поле для указания количества задействованных ставок сотрудников университета являющихся совместителями.
  7. Поле «Лекции (ак. часов)» — поле для указания академических часов.
  8. Поле «Практика (ак. часов)» — поле для указания академических часов.
  9. Поле «См. работа (ак. часов)» — поле для указания академических часов.
  10. Поле «Нормативные документы» — поле позволяющее загрузить с компьютера пользователя файл содержащий внутреннюю нормативную документацию университета по реализуемой образовательной программе.
  11. Поле «Учебно методические результаты» — поле для описания учебно методических результатов.
  12. Чекбок «Запланировано создание новых образовательных пространств» — позволяет обозначить пользователю, что в рамках реализации ОП запланировано создание новых образовательных пространств.
  13. Кнопка «Сохранить» — по нажатии указанные данные образовательной программы сохраняются, а пользователь будет направлен на страницу с информацией о ОП.

* - при указании ставок суммируйте именно ставки, а не сотрудников задействованных в реализации образовательной программы (например если у нас задействован 1 сотрудник со ставкой 1 и 2 сотрудника со ставками 0,5, указывать необходимо 2 в соответствующем поле формы)

Связь с научными проектами​

Вкладка «Связь с научными проектами» уникальная для ОП. Все остальные вкладки совпадают между НП и ОП.

Рисунок № 8 — Элементы интерфейса вкладки «Связь с научными проектами»

На вкладке доступны следующие элементы интерфейса:

  1. Иконка «Удалить» — удаляет связь научный проекта с образовательной программой (сам НП не удаляется).
  2. Поле «Научный проект» — позволяет выбрать один из внесённых ранее научных проектов.
  3. Кнопка «Добавить» — по нажатии на кнопку выбранный НП в поле «Научный проект» добавляется в список и связывается с данной образовательной программой.

Общие элементы интерфейса​

Партнёры​

Рисунок № 9 — Элементы интерфейса вкладки «Партнёры»

На вкладке доступны следующие элементы интерфейса:

  1. Иконка «Удалить» — удаляет связь партнёра с НП или ОП (сам партнёр не удаляется).

  3. Кнопка «Добавить» — по нажатии на кнопку выбранный партнёр в поле «Высокотехнологичная компания (партнёр)» добавляется в список и связывается с НП или ОП.

Оборудование и ПО​

Рисунок № 10 — Элементы интерфейса вкладки «Оборудование и ПО»

На вкладке доступны следующие элементы интерфейса:

  1. Иконка «Редактировать» — по нажатии пользователь переходит на страницу редактирования информации об оборудовании или программном обеспечении.
  2. Иконка «Удалить» — удаляет оборудование или программное обеспечение из НП или ОП.

Добавление/редактирование оборудования/ПО​

Рисунок № 11 — Страница добавления/редактирования оборудования/ПО

На странице доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии на кнопку пользователь перейдёт к НП или ОП (зависит от того, куда в данный момент добавляется оборудование или ПО).
  2. Поле «Название» — текстовое поле для указания названия.
  3. Поле «Тип» — выбор из «оборудования» или «программного обеспечения».
  4. Поле «Наличие» — выбор из «в наличии», «будет приобретено» или «будет арендовано».
  5. Поле «Владелец» — университет или один из партнёров НП или ОП (зависит от того, куда в данный момент добавляется оборудование или ПО).
  6. Кнопка «Создать» / «Сохранить» — по нажатии на кнопку создаётся новое оборудование или ПО или сохраняются данные уже существующего (завист от того, создаётся или редактируется оборудование или ПО).

Кадровое обеспечение​

Рисунок № 12 — Элементы интерфейса вкладки «Кадровое обеспечение»

Во вкладке доступны следующие элементы интерфейса:

  1. Иконка «Просмотр» — по нажатии пользователю открывается информация с данными пользователя. Интерфейс с данными человека совпадает с интерфейсом добавления и редактирования пользователя, с той лишь разницей, что все поля с данными заблокированы для редактирования.
  2. Иконка «Редактировать» — по нажатии пользователь открывается интерфейс для редактиварония пользователя.
  3. Иконка «Удалить» — удаляет пользователя из НП или ОП (удаление данных пользователя не происходит).
  4. Кнопка «Добавить» — по нажатии пользователь переходит на страницу добавления кадров.

Добавление кадров​

Рисунок № 13 — Добавление кадров из числа сотрудников университета

На странице доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии на кнопку пользователь перейдёт к кадровому обеспечению.
  2. Блок «Тип пользователя» — позволяет выбрать является ли новый член кадрового обеспечения сотрудников университета или сотрудников одного из партнёров, участвующих в реализации НП или ОП.
  3. Кнопка «Добавить нового пользователя» — по нажатии откроется интерфейс добавление пользователя.
  4. Поле «Роль» — текстовое поле для указания роли в рамках НП или ОП.
  5. Поле «Пользователь» — выбор конкретного пользователя. Список зависит от выбранного ранее в блоке «Тип пользователя» (пункт № 2 данного списка).
  6. Кнопка «Сохранить» — сохраняет введённые данные и возвращает текущего пользователя на предыдущую страницу.

Рисунок № 14 — Добавление кадров из числа сотрудников партнёров

На странице доступны следующие элементы интерфейса:

  1. Кнопка «Назад» — по нажатии на кнопку пользователь перейдёт к кадровому обеспечению.
  2. Блок «Тип пользователя» — позволяет выбрать является ли новый член кадрового обеспечения сотрудников университета или сотрудников одного из партнёров, участвующих в реализации НП или ОП.
  3. Поле «Организация» — позволяет выбрать партнёра привязанного к НП или ОП (более подробнее о привязке партнёров читайте в соответствующем подразделе).
  4. Кнопка «Добавить нового пользователя» — по нажатии откроется интерфейс добавление пользователя.
  5. Поле «Роль» — текстовое поле для указания роли в рамках НП или ОП.
  6. Поле «Пользователь» — выбор конкретного пользователя. Список зависит от выбранного ранее в блоке «Тип пользователя» (пункт № 2 данного списка).
  7. Кнопка «Сохранить» — сохраняет введённые данные и возвращает текущего пользователя на предыдущую страницу.
Обратите внимание

В поле «Организация», при добавлении из числа сотрудников партнёров, содержит список организаций, которые привязаны к конкретному научному проекту или образовательной программе (более подробнее о привязке партнёров читайте в соответствующем подразделе), а не все организации введённые в разделе «Партнёры».

  11. Поле «Адрес эл. почты» — текстовое поле для указания
  12. Поле «Контактный телефон» — текстовое поле для указания
  13. Поле «Учёное звание» — текстовое поле для указания адреса электронной почты.
  14. Поле «Учёная степень» — поле для указания учёной степени.
  15. Поле «ORCID» — текстовое поле для указания ORCID.
  16. Поле «Scopus ID» — текстовое поле для указания Scopus ID автора.
  17. Поле «Researcher ID» — текстовое поле для указания Researcher ID автора.
  18. Поле «РИНЦ (authorId)» — текстовое поле для указания РИНЦ (authorId) автора
  19. Поле «Согласие на обработку ПД» — поле позволяющее загрузить с компьютера пользователя файл с копией согласия на обработку персональных данных.
  20. Кнопка «Сохранить» — сохраняет введённые данные о пользователи и закрывает текущий интерфейс.
